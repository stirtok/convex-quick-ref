Use ⌘+f on a Mac / ⊞+f on Windows to search for some text in Convex's standard libraries.[1] Or, explore through the expandable namespace list.

1. Generated from Convex’s 20th August 2025 commit 3e53bb8.

convex.core

!=

Tests if arguments are not equal in numerical value. Inverse of ==.

Signature

Params: [& xs] Return: & xs

Examples
(!= 2 2.0)

*

Multiplies numeric arguments. Result will be a Double if any arguments are floating point values, otherwise it will be a long.

Signature

Params: [& xs]

Examples
(* 1 2 3 4 5)

*address*

Returns the address of the current account (user address in regular transaction, actor address in actor calls).

Examples
(address? *address*)

*balance*

Returns the available balance of the current account (in Convex coins).
The available balance excludes reserved balance for transaction execution, so this number may be somewhat less than the total account balance during transaction execution.

Examples
*balance*

*caller*

During an actor call, returns the address of the account doing the call. Returns nil otherwise.

Examples
*caller*

*controller*

Returns the controller of the current account.

Examples
*controller*

*depth*

Returns the CVM execution stack depth, at the point the *depth* operation is executed. If the depth becomes too deep, the transaction will fail with a :DEPTH exception.
In most cases, the allowable depth should be sufficient.

Examples
*depth*

*holdings*

Returns the holdings Index for this account.
Holdings are data values controlled by other accounts (usually actors). They can be used to indicate that an account may have special rights or asset holdings with respect to a specific actor, for instance.
Holdings should NOT be trusted if they are provided by an untrusted account.
See get-holding, set-holding.

Examples
*holdings*

*initial-expander*

Initial expander used to expand forms, before compilation.

Signature

Params: [form cont]

Examples
(expand '(if 1 2 3) *initial-expander*)

*juice*

Returns the amount of execution juice consumed at this point of the current transaction.
Juice is required for every CVM operation executed, and the transaction will fail immediately with a :JUICE error if an attempt is made to consume juice beyond this value.

Examples
*juice*

*juice-limit*

Returns the limit of execution juice available.
Juice is required for every CVM operation executed, and the transaction will fail immediately with a :JUICE error if an attempt is made to consume juice beyond this value.

Examples
*juice-limit*

*key*

Returns the public key for this account. Returns nil in the case of an actor since actors are accounts without keys.

Examples
*key*

*lang*

Advanced feature. Language front-end function.
If set to a function via def, will be called with the code for each transaction instead of delegating to normal eval behavior.
Pre-compiled operations (see compile) bypass this language setting.

Examples
(def *lang* (fn [trx] (str trx)))

*location*

Returns the [block-index transaction-index] location of the current transaction.

Examples
*location*

*memory*

Returns the current memory allowance for this account. May be zero - in which case any new memory allocations will be charged at the current memory exchange pool price.

Examples
*memory*

*memory-price*

Returns the current memory price, as a double value

Examples
*memory-price*

*nop*

Does nothing.

Examples
*nop*

*offer*

Returns the amount of native coin offered by *caller* during a call. See call. Will usually be zero, unless the caller has included an offer with a 'call' expression.

Examples
*offer*

*origin*

Similar to *caller* but returns the address of the account that initially signed this transaction.
In a chain of calls, this address is the very first link.
Usually, should NOT be used for access control, since a rogue actor can potentially trick a user into creating a transaction that allows code to be indirectly executed. Consider using *caller* for access control instead.

Examples
*origin*

*parent*

Returns the parent of the current account. Determines the default location for symbolic lookup.

Examples
*parent*

*peer*

Returns the peer responsible for the current Block, or null if outside a block (e.g. query).

Examples
*peer*

*registry*

Address of the Convex registry actor.

Examples
(call *registry* (register {:name "My name"}))

*result*

Returns the result of the last CVM operation executed. Can be used, in some cases, to access the value of the previous expression. Will be nil for new transactions, or at the start of an actor call.

Examples
(do 1 *result*)

*sequence*

Returns the sequence number for the current transaction. The sequence number is the number of transactions executed on this account *including* the current transaction / query. The next valid transction must increment *sequence* by one.

Examples
*sequence*

*signer*

Returns the public key used to sign the current transaction.

Examples
*signer*

*state*

Returns the current CVM state record. This is a large object, and should normally only be used temporarily to look up relevant values.

Examples
(keys *state*)
(get-in *state* [:accounts *address* :balance])

*timestamp*

Returns the current timestamp.
The timestamp is a Long value that is equal to the greatest timestamp of any block executed (including the current block).
A timestamp can be interpreted as the number of milliseconds since January 1, 1970, 00:00:00 GMT. The block timestamp should always be less than or equal to the Unix timestamp of peers that are in consensus.
Commonly used with schedule.

Examples
*timestamp*

accept

Used during a call, accepts offered coins up to the amount of *offer* from *caller*. Returns the amount accepted if successful.
Amount must cast to Long. If successful, the amount will be added immediately to the *balance* of the current *address*.
This is the recommended way of transferring balance between actors, as it requires a positive action to confirm receipt.

Signature

Params: [amount] Return: amount

Errors

ARGUMENT: If accepted amount is negative

CAST: If accepted amount is not a long

STATE: If ´*caller*´ has not offered sufficient coins to fulfil the offer

Examples
(accept *offer*)

and

Executes expressions in sequence, returning the first falsey value (false or nil), or the last value otherwise.
Does not evaluate later expressions, so can be used to short circuit execution.
Returns true with no expressions present.

Examples
